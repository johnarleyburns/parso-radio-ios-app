Build notes grounded in the real catalog

Every onboarding chip maps to a real collection id (from default_collections.json) or a real subject/creator seed, so a brand-new user gets a working shelf with no backfill. Selecting chips writes weighted terms into the durable taste profile, then the same candidate generator that powers history-based picks runs over those seeds.

Behavior

  • Forward-only, no backfill. Existing history is not re-scanned. Profile accumulates from the next play/favorite onward; onboarding fills the day-one gap.
  • Seed weight < a real play. Chips seed at ~1.5–2× a single play, and decay (TAU≈21d) lets actual listening overtake the onboarding priors within a few weeks.
  • Rail, not a single card. Replace the "Featured today" Made-for-You FeaturedCard with a horizontal section mirroring HomeTopSection (Jump Back In). Cards are individual tracks.
  • Minimum 10 picks. Controller returns ≥10; if EXPLOIT/EXPLORE come up short, top up from SERENDIPITY before falling back. The rail is hidden only if the profile is truly empty (skipped onboarding + no plays).
  • Tap a card = play that track, then enqueue the remaining picks as up-next, so it still behaves like a station once entered.
  • Still never repeats. Excludes played ∪ favorited ∪ recently-surfaced; daily date-seed keeps the shelf stable within a day and new across days.
