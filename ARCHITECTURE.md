# Architecture

SwiftUI + MVVM, iOS 17+, Universal (iPhone/iPad). No local toolchain — see
`TESTING.md`. Xcode project generated by XcodeGen from `project.yml`.

## Layers

```
Views (SwiftUI)  ──>  ViewModels (@MainActor, ObservableObject)  ──>  Services
```

### Views (`ParsoRadio/Views/`)
- **iPodView** — the device shell: screen panel + `ClickWheel`. Renders
  artwork, the `ProceduralVisualizerView` (no-art fallback), or a looping
  ambient `LoopingVideoView`. Hosts the combined Track-Info/Options sheet.
- **MainMenuView** — channels by category; playlists (tap = Resume, ••• =
  detail). **SearchView** — query + result actions + recent history.
  **PlaylistDetailView** — play/shuffle/resume/edit. **AboutView** — privacy,
  credits, Copyright & DMCA. **TermsView** — first-run EULA gate.

### ViewModels (`ParsoRadio/ViewModels/`)
- **PlayerViewModel** — the brain. Channel/playlist load, the queue cursor,
  `playTrack`, advance/back, resume (per-channel & per-playlist position via
  the positions table, playlist key `playlist:<id>`), artwork (cleared
  synchronously per track + stale-guarded), the multi-file probe
  (`resolveItemParts`), `addEntireItemToPlaylist`, News "jump to newest on
  re-entry", and invisible min-duration pre-screening (`assetDuration`).
- **PlaylistViewModel** — playlists CRUD, ordered bulk add, reorder, favorites.
- **SearchViewModel** — debounced IA search, recent history, item-kind
  classification (track/album/book), result ranking.

### Services (`ParsoRadio/Core/Services/`)
- **API/InternetArchiveService** — IA Solr search; `fetchTracks(iaQuery:
  matchTags:)` for registry channels (stamps tracks); `fetchTracksForIdentifier`
  picks a SINGLE audio format in natural chapter order; `search`/`itemInfo`
  for the Search screen.
- **API/FMAService** — scrapes freemusicarchive.org genre pages.
- **API/OxfordLecturesService** — podcasts.ox.ac.uk units → series → RSS.
- **API/PodcastRSSService** — News RSS feeds.
- **API/AmbientStaticService** — hardcoded ambient tracks; bundle-first
  resolver for offline gapless loop audio/video.
- **Playback/QueueManager** — per-channel recency, deterministic
  `usesShuffle`, weighted random (album-boosted), podcast newest-first,
  LibriVox part navigation. **Channel pools exclude `partNumber > 1`** so a
  channel only ever offers a book's first track.
- **Playback/AudioPlayerService** — `@MainActor` AVPlayer wrapper; AVPlayerLooper
  for ambient; large `preferredForwardBufferDuration` for poor connectivity;
  Now Playing / remote commands; interruption & route handling.
- **Storage/DatabaseService** — async SQLite (`withCheckedContinuation`):
  tracks, playlists (+`sort_order`), positions, play history; idempotent
  schema migrations; `is_multi_part`, parent-identifier queries.
- **Metadata/ArtworkService**, **Download/OfflineDownloadService**,
  **IAQueryRegistry** (loads `ia_queries.json`).

### Models (`ParsoRadio/Core/Models/`)
`Track` (incl. `partNumber`/`parentIdentifier`/`isMultiPart`/`recordingDate`),
`Channel` (registry entry via `iaQueryEntry`; `minTrackDuration`;
`Channel.stampToken`), `Playlist`.

## Key design decisions

- **Pure-Lucene registry channels.** One curl-verified query per channel in
  `ia_queries.json`; NO code-side license/category filtering; `sort=random`
  every load. Curated/explicit-allowlist channels (Ancient Greece, Great/
  Greater Books) enumerate authors/works explicitly to avoid bulk-subject
  noise; English/Greek-only where relevant. IA caps `q` length (~1700 chars),
  so allowlists are bounded curated subsets.
- **Stamp isolation.** Registry fetches stamp each track with
  `pmreg::<channelId>`; `Channel.matches` checks the stamp, so the shared DB
  never cross-contaminates channels even with overlapping queries.
- **Book channels = first track only.** Probed chapters are saved (for
  playlist add / parent lookup) but NOT channel-stamped, and QueueManager
  drops `partNumber > 1` — the channel rotates book→book; "Add Book to
  Playlist" adds every chapter in order.
- **Resume.** Channel & playlist positions persist in one positions table;
  spoken-word resumes at the exact offset; News jumps to the newest episode
  on re-entry if one is newer than last played.
- **Min-duration channels** (Children's Songs) pre-screen candidate duration
  with `AVURLAsset.load(.duration)` BEFORE revealing — too-short tracks are
  invisible (spinner only), never shown or heard.
- **Ambient loops.** Bundled CC0 PCM WAV (seamless crossfade-built) + muted
  looping video; both freeze when audio is paused.
- **No audio-tap visualizer.** `ProceduralVisualizerView` is a seeded
  `TimelineView`/`Canvas` animation (deterministic per-track palette) — no
  `MTAudioProcessingTap` (which previously caused a device crash).

## Persistence

SQLite at `Documents/parso_radio.sqlite`. Tracks cached and evicted (>5k rows,
>30 days, non-local). Positions keyed by channel id or `playlist:<id>`.
Search history & settings in `UserDefaults` (declared in the privacy manifest).
