# Parso Radio

An iPod-inspired iOS internet-radio player for **public-domain and Creative
Commons** music, audiobooks, and lectures. No ads, no accounts, no tracking —
the source and license are shown for every track.

## Features

- **Click-wheel UI** — tap MENU to pick a channel; tap the wheel to
  play/pause, skip, or go back; drag the wheel to seek.
- **Curated channels** — Spanish Guitar, Chamber Music, Symphony, Piano Hour,
  Café Lento, 78 RPM, Ancient Greece, Great Books, Greater Books, Children's
  Books, Children's Songs, and more.
- **LibriVox audiobooks** (21 genre channels), **Oxford University
  lectures**, and **News** podcast channels — the public-library / public-radio
  core (the contemporary/FMA genre channels were dropped to sharpen this focus).
- **Ambient loops** — Flowing Water / Rainy Day / Ocean Waves: bundled,
  gapless, offline CC0 audio with a looping video backdrop.
- **Search** the Internet Archive; tap a result to Play, Add to Playlist, or
  Add the whole Book/Album (incl. straight to a new playlist).
- **Playlists** with reorder/delete and **exact-position Resume** (critical
  for long multi-part audiobooks).
- **Offline** downloads; resilient buffering for poor connectivity; lock-screen
  / CarPlay transport.
- **Procedural visualizer** when a track has no artwork; animations pause with
  the audio.

## Channel model

Most channels are **pure-Lucene Internet Archive registry channels**: a single
hand-tuned `archive.org/advancedsearch` query in
`ParsoRadio/Resources/ia_queries.json`, played in server-side random order. A
per-channel namespaced **stamp** isolates each channel's tracks in the shared
local DB. Book channels only ever play a book's **first track**; the user adds
the whole book to a playlist if they want it. See `ARCHITECTURE.md`.

## Build & release

This project has **no local Swift/iOS toolchain** — all builds run on GitHub
Actions (macOS runners) via `.github/workflows/ios.yml`:

1. Unit tests (`ParsoMusicTests`) + live-IA integration tests
   (`ParsoMusicIntegrationTests`).
2. Archive (Release) → export IPA → upload to TestFlight.

The Xcode project is generated by **XcodeGen** from `project.yml` (anything
under `ParsoRadio/` is bundled automatically). The build number is the CI run
number; the marketing version auto-increments per push (`2.0.x`). Details and
the remaining manual App Store steps are in `APPSTORE.md` / `APPSTORE-MANUAL.md`.

## Content & licensing

All streamed audio is public-domain or Creative Commons; per-track license is
shown in-app. Bundled ambient audio is CC0 (`ParsoRadio/Resources/Audio/
ATTRIBUTION.md`). In-app **Copyright & DMCA** reporting is under About. Privacy
policy: <https://parso.guru/parso-radio-privacy>. No data is collected.
